# Colors

The full Crash Override palette. It is built in layers: the **official brand colours**
(the literal Brand Book values — five colours, no blue), the **brand scales** that ramp
each accent for hover/text/tint use, a set of **status, terminal, and code** utility hues,
the **capability palette** (`--chart-1…5`, colour-coding the product spine), and the
**semantic shadcn tokens** that every component consumes.

Every fixed token below is painted from its literal value and shows its **token name +
hex/value**, cross-checked against `tokens/colors.css`. The capability and semantic tokens
re-resolve per theme — flip the **theme toggle in the toolbar** and those swatches recolour
live.

## Semantic tokens (shadcn bridge)

The exact CSS variables shadcn/ui expects, mapped to the brand. Painted with
`bg-[var(--token)]` so each chip **reflects the active theme** — toggle dark/light in the
toolbar to watch them flip. `--primary` is Atomic lime on dark, Fandango purple on light.

> **Why oklch?** The semantic tokens are defined in `oklch(…)` for perceptually uniform
> lightness across both themes. You never reference them directly — you use the Tailwind
> utilities (`bg-primary`, `text-muted-foreground`, `border-border`) or the component
> props, and they resolve correctly in whichever theme is active.
